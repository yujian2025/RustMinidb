🔌 REST API 接口文档

RustMinidb 提供以下 HTTP API 接口，所有请求和响应均为 JSON 格式。

📊 服务器信息

GET/v1/health
健康检查，返回服务器状态、版本、运行时间、表数、当前数据库
curl http://localhost:8080/v1/health

🗄️ 数据库管理

GET/v1/databases
列出所有数据库文件，标记当前激活的数据库
curl http://localhost:8080/v1/databases
POST/v1/databases/create
创建新数据库并自动切换到该数据库
curl -X POST http://localhost:8080/v1/databases/create \
-H "Content-Type: application/json" \
-d '{"name":"myapp"}'
POST/v1/databases/switch
切换到指定数据库（不存在时可通过 create:true 自动创建）
curl -X POST http://localhost:8080/v1/databases/switch \
-H "Content-Type: application/json" \
-d '{"name":"myapp"}'

📋 表管理

GET/v1/tables
列出当前数据库中的所有表
curl http://localhost:8080/v1/tables
GET/v1/schema/{table}
查看指定表的列名、类型、主键、非空约束
curl http://localhost:8080/v1/schema/sensors

📝 SQL 执行

POST/v1/query
执行 SQL 语句（CREATE / INSERT / SELECT / UPDATE / DELETE / DROP）
# 创建表
curl -X POST http://localhost:8080/v1/query \
-H "Content-Type: application/json" \
-d '{"sql":"CREATE TABLE users (id INT PRIMARY KEY, name TEXT, age INT)"}'

# 插入数据
curl -X POST http://localhost:8080/v1/query \
-H "Content-Type: application/json" \
-d '{"sql":"INSERT INTO users VALUES (1, '\''Alice'\'', 30)"}'

# 查询
curl -s http://localhost:8080/v1/query \
-H "Content-Type: application/json" \
-d '{"sql":"SELECT * FROM users WHERE age > 20 ORDER BY name LIMIT 10"}'

# 更新
curl -X POST http://localhost:8080/v1/query \
-H "Content-Type: application/json" \
-d '{"sql":"UPDATE users SET age = 31 WHERE id = 1"}'

# 删除
curl -X POST http://localhost:8080/v1/query \
-H "Content-Type: application/json" \
-d '{"sql":"DELETE FROM users WHERE id = 2"}'

📥 数据导入

POST/v1/import
批量导入 JSON 数组格式的数据到指定表
curl -X POST http://localhost:8080/v1/import \
-H "Content-Type: application/json" \
-d '{"table":"users","data":[{"id":3,"name":"Charlie","age":28},{"id":4,"name":"Diana","age":35}]}'

📌 通用说明

所有 API 统一返回格式：
{
"success": true/false,
"data": { "columns":[...], "rows":[[...],...], "rowsAffected":N, "timeMs":1.23 },
"error": { "code":"ERROR_CODE", "message":"错误描述" }
}

错误码: PARSE_ERROR, TABLE_NOT_FOUND, PRIMARY_KEY_CONFLICT, TYPE_MISMATCH, VALIDATION_ERROR
